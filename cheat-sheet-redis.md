## Шпаркалка по Redis

### Redis (Remote Dictionary Server)

Что это?
Это высокопроизводительное хранилище данных «ключ-значение», которое держит все данные в оперативной памяти (с возможностью сохранения на диск). Работает как сетевой сервер, к которому подключаются клиенты. Главные фишки: скорость (микросекунды на запрос), богатые структуры данных (строки, списки, множества, хеши, сортированные множества и др.) и встроенные механизмы pub/sub, транзакций, истечения срока жизни ключей.
#### Распространённые паттерны применения

- Кэширование — хранение результатов тяжёлых запросов к БД, HTML-фрагментов, сессий. Данные быстро отдаются, не нагружая основную базу.
- Хранилище сессий — сессии пользователей веб-приложений живут в Redis с TTL (временем жизни), легко масштабируются между серверами.
- Очереди задач (брокер сообщений) — класть задачи в список, воркеры разбирают. Именно так с Redis работает Celery (хотя Celery умеет и другие брокеры).
- Лидерборды (таблицы рекордов) — сортированные множества идеально подходят для рейтингов в реальном времени.
- Pub/Sub — уведомления в реальном времени: чаты, оповещения, инвалидация кэша по событию.
- Ограничитель частоты (Rate Limiter) — подсчёт запросов от пользователя с истечением счётчика.
- Блокировки (Distributed locks) — гарантия, что операция выполняется только одним процессом в кластере (через SETNX).

#### Как работать с Redis из командной строки

Основная утилита — `redis-cli`. Если Redis запущен локально на порту 6379, просто введи `redis-cli`.  

|Команда|Назначение|
|-----|-----|
|**Строки**||
|`PING`|Проверить соединение (вернёт `PONG`)|
|`SET key value`|Записать строку|
|`GET key`|Прочитать строку|
|`DEL key1 key2 ...`|Удалить ключи|
|`EXISTS key`|Проверить существование|
|`EXPIRE key seconds`|Установить TTL в секундах|
|`TTL key`|Посмотреть оставшийся TTL (-1 — вечный, -2 — нет ключа)|
|`KEYS pattern`|Найти ключи по шаблону (например, KEYS user:*). **Осторожно в продакшене!**|
|`SCAN cursor`|Безопасная альтернатива `KEYS` для обхода всех ключей|
|`TYPE key`|Тип данных ключа|
|**Строки**||
|`INCR key`|Увеличить число на 1|
|`DECR key`|Уменьшить на 1|
|`APPEND key string`|Дописать строку|
|**Списки**||
|`LPUSH list val` / `RPUSH list val`|Добавить в начало/конец|
|`LPOP list` / `RPOP list`|Извлечь с начала/конца|
|`LRANGE list 0 -1`|Все элементы|
|**Хеши**||
|`HSET hash field val`|Установить поле|
|`HGET hash field`|Получить значение поля|
|`HGETALL hash`|Все поля и значения|
|**Множества**||
|`SADD set member`|Добавить элемент|
|`SMEMBERS set`|Все элементы|
|`SISMEMBER set member`|Проверить наличие|
|**Множества**||
|`ZADD zset score member`|Добавить с рейтингом|
|`ZRANGE zset 0 -1`|По возрастанию score|
|`ZREVRANGE zset 0 -1 WITHSCORES`|Лидеры по убыванию с баллами|
|**Pub/Sub**||
|`SUBSCRIBE channel`|Подписаться|
|`PUBLISH channel msg`|Опубликовать сообщение|
|**Мониторинг и администрирование**||
|`INFO`|Статистика сервера (память, клиенты, ключи)|
|`INFO memory`|Только статистика по памяти|
|`MONITOR`|Поток всех команд, выполняемых на сервере (дебаг)|
|`CLIENT LIST`|Список подключённых клиентов|
|`CONFIG GET parameter`|Посмотреть конфиг (например, `maxmemory`)|
|`CONFIG SET param val`|Изменить на лету|
|`SAVE`|Принудительно сохранить дамп на диск|
|`BGSAVE`|Асинхронное сохранение|
|`LASTSAVE`|Unix timestamp последнего сохранения|
|`FLUSHDB`|Очистить текущую БД (осторожно!)|
|`FLUSHALL`|Очистить все БД|
|`DBSIZE`|Количество ключей в текущей БД|
|`SELECT index`|Переключиться на БД (по умолчанию 0, всего 16)|

Пример использования в терминале:
```bash
redis-cli
127.0.0.1:6379> SET mykey "Hello"
127.0.0.1:6379> GET mykey
127.0.0.1:6379> EXPIRE mykey 60
127.0.0.1:6379> TTL mykey
```
Для подключения к удалённому серверу:
```bash
redis-cli -h 10.0.0.5 -p 6380 -a password
```
