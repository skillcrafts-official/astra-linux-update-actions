## Шпаркалга по Celery
### Celery

Что это?
Асинхронная распределённая очередь задач на Python. Celery принимает задания от приложения, ставит их в очередь через брокера сообщений (Redis, RabbitMQ и др.) и выполняет в отдельных процессах-воркерах. Результаты могут сохраняться в backend (Redis, БД). Поддерживает периодические задачи (по расписанию), группировку, цепочки, повтор при ошибках и мониторинг.
#### Распространённые паттерны применения

- Фоновые задачи — отправка email, генерация отчётов, обработка изображений, не блокируя HTTP-ответ пользователю.
- Периодические задачи — cron-подобные: очистка старых сессий каждую ночь, синхронизация с внешним API раз в час.
- Цепочки задач (chords/chains) — последовательная обработка: загрузили файл -> проверили вирусы -> создали превью -> уведомили пользователя.
- Группировка задач (groups) — параллельное выполнение (например, отправить push-уведомления тысяче устройств).
- Долгие операции с прогрессом — задача обновляет статус, а клиент опрашивает его через хранилище результатов.
- Обработка событий из очереди — воркеры слушают очередь Redis, выполняют работу при поступлении сообщения.

#### Как работать с Celery из командной строки

Главная утилита — команда `celery`. Предполагаем, что у нас есть приложение Celery, 
определённое в Python-модуле, например, `tasks.py` или в пакете `myproject`, 
и экземпляр приложения называется `app`. 
Тогда базовый вызов выглядит как `celery -A <имя модуля или пакета> <команда>`.

Шпаргалка по `celery`
|Команда|Назначение|
|--|--|
|Запуск воркера||
|`celery -A tasks worker --loglevel=info`|Запустить воркер (слушает задачи и выполняет)|
|`celery -A tasks worker -Q queue1,queue2`|Воркер слушает конкретные очереди|
|`celery -A tasks worker --concurrency=8`|Количество процессов (по умолчанию — число ядер)|
|`celery -A tasks worker -n worker1@%h`|Задать имя воркера (`%h` — hostname)|
|Просмотр активных и зарегистрированных задач||
|`celery -A tasks inspect active`|Активные задачи на всех воркерах|
|`celery -A tasks inspect scheduled`|Запланированные (ETA, countdown)|
|`celery -A tasks inspect registered`|Все зарегистрированные имена задач|
|`celery -A tasks inspect stats`|Статистика воркеров|
|`celery -A tasks inspect reserved`|Задачи, взятые воркером, но ещё не выполненные|
|Управление воркерами||
|`celery -A tasks control shutdown`|Плавно остановить всех воркеров (завершат текущие задачи)|
|`celery -A tasks control pool_restart`|Перезапустить пул процессов|
|`celery -A tasks control enable_events`|Включить отправку событий для мониторинга (нужно для Flower)|
|Периодические задачи||
|`celery -A tasks beat`|Запуск планировщика beat (рассылает задачи по расписанию)|
|`celery -A tasks beat --loglevel=info --scheduler django_celery_beat.schedulers:DatabaseScheduler`|Beat с хранением расписания в БД (Django)|
|Очистка очередей||
|`celery -A tasks purge`|Удалить все сообщения из всех очередей (осторожно!)|
|Просмотр результатов||
|`celery -A tasks result <task_id>`|Получить результат задачи по ID|
|Мониторинг (цветной интерфейс)||
|`celery -A tasks flower`|Запускает Flower (веб-интерфейс мониторинга, порт 5555)|

#### Важные замечания к командам:

- Все инспекционные команды (`inspect active` и т.д.) работают только если воркеры запущены и
включены события (`-E` при запуске worker или `control enable_events`).
- `purge` удаляет задачи из очереди, но не отменяет уже выполняющиеся.
- Чтобы отправить саму задачу вручную из командной строки, используют самописные скрипты или `celery call`:

```bash
celery -A tasks call myapp.tasks.send_email --args='["user@example.com"]' --kwargs='{}'
```

### Типичный рабочий процесс (сценарий)

- В терминале 1 поднимаем Redis (обычно он уже запущен как сервис).
- В терминале 2 запускаем Celery воркер: `celery -A myapp worker -l info`.
- В коде приложения (или в shell) вызываем задачу: `send_mail.delay(...)`. 
Задача попадает в Redis-очередь, воркер подхватывает и выполняет.
- Для периодики запускаем `celery -A myapp beat` в третьем терминале.
- Смотрим статус: `celery -A myapp inspect active`, `redis-cli monitor`, или веб-морду `flower`.
